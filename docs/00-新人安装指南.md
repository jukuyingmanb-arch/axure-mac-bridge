# 新人安装指南（零基础逐步版）

> 完全没用过命令行也能跟着走。每一步都告诉你**该输什么**、**应该看到什么**、**出错怎么办**。
> 全程大约 10 分钟，其中等你点几次"允许"。

---

## 你需要先确认的 3 件事

| 检查项 | 怎么确认 | 没有怎么办 |
|---|---|---|
| 1. 这是一台 Mac | 屏幕左上角有 🍎 | 这工具只能在 Mac 上跑 |
| 2. 装了 Axure RP 11 | 启动台里找得到 Axure | 去 axure.com 下载安装 |
| 3. 装了 Node.js | 见下方"步骤 0" | 见下方"步骤 0" |

---

## 步骤 0：装 Node.js（如果已装可跳过）

### 怎么判断装没装

打开**终端**（按 `⌘ + 空格`，输入 `终端` 或 `Terminal`，回车），粘贴这行后回车：

```bash
node -v
```

- 看到 `v18.x.x` 或更高 → ✅ 已装，跳到步骤 1
- 看到 `command not found` → 没装，继续往下

### 安装 Node.js

最简单：去 https://nodejs.org/ 下载 **LTS 版本**，双击 `.pkg` 一路下一步。

装完**关掉终端再重开**，重新 `node -v` 确认。

---

## 步骤 1：拿到代码

### 方式 A：如果别人给了你 .zip 安装包

1. 双击解压 `axure-mac-bridge.zip`
2. 把解压出的 `axure-mac-bridge` 文件夹拖到桌面

### 方式 B：从 GitHub clone（如果给的是 git 链接）

终端里粘贴（把 `<链接>` 换成实际地址）：

```bash
cd ~/Desktop
git clone <链接> axure-mac-bridge
```

> 如果提示 `git: command not found`，终端会自动弹窗让你装"命令行开发者工具"，点"安装"等几分钟即可。

---

## 步骤 2：运行安装脚本

终端里**逐行**粘贴回车：

```bash
cd ~/Desktop/axure-mac-bridge
./INSTALL.sh
```

> 如果你的文件夹在别处，把 `~/Desktop/axure-mac-bridge` 换成实际路径。
> 不确定路径？把文件夹拖进终端窗口，它会自动填路径，删掉 `cd ` 后面原来的部分即可。

**你应该看到**：

```
========================================
  axure-mac-bridge installer
========================================

[1/5] 检查前置条件...
  ✅ macOS
  ✅ Node.js v20.x.x
  ✅ Axure RP 11

[2/5] 安装 AxureBridge.app...
  ✅ 拷到 /Users/你/Library/Application Support/axure-bridge/AxureBridge.app

[3/5] 注册到 LaunchServices...
  ✅ 已注册

[4/5] 建 ~/Desktop/原型/ 目录...
  ✅ /Users/你/Desktop/原型

[5/5] 完成。
```

**出错怎么办**：
- `permission denied: ./INSTALL.sh` → 先跑 `chmod +x INSTALL.sh` 再重试
- `❌ 缺 Node.js` → 回步骤 0
- `❌ 找不到 AxureBridge.app` → 你 `cd` 错目录了，确认在解压出来的文件夹里

---

## 步骤 3：第一次生成原型

确保 **Axure RP 11 已经打开，并新建了一个空白文件**（菜单 File → New）。

终端里粘贴：

```bash
cd ~/Desktop/axure-mac-bridge/svg-generator
node examples/feedback-page.mjs
```

**你应该看到**：

```
✅ Wrote SVG: /Users/你/Desktop/原型/feedback-prototype.svg
   Dimensions: 1007 x 2976
   Chars: 24373
```

这一步生成了一个 SVG 文件，放在 `~/Desktop/原型/`。

---

## 步骤 4：预览 + 准备导入

终端里粘贴这两行：

```bash
open -a Safari ~/Desktop/原型/feedback-prototype.svg

open -a "$HOME/Library/Application Support/axure-bridge/AxureBridge.app" \
     --args "$HOME/Desktop/原型/feedback-prototype.svg"
```

- 第一行：Safari 打开预览，你能先看到 3 个反馈页面
- 第二行：触发 AxureBridge

### 第一次跑会弹权限对话框（重点！）

会**依次**弹 2~3 个：

```
┌────────────────────────────────────┐
│  "AxureBridge" 想要控制              │
│  "系统事件(System Events)"           │
│                                      │
│        [不允许]    [允许]   ← 点这个  │
└────────────────────────────────────┘
```

每个都点 **允许**：
1. AxureBridge 想控制 **系统事件** → 允许
2. AxureBridge 想控制 **Axure RP 11** → 允许
3. AxureBridge 想控制 **访达(Finder)** → 允许

> ⚠️ 对话框有时藏在其他窗口后面。如果终端跑完没反应，按 `⌘ + Tab` 切窗口找一下。

授权完之后，**访达会自动打开 `原型` 文件夹并高亮选中那个 SVG，Axure 也会跳到前台**。

---

## 步骤 5：最后 1 秒——手动拖

> 这一步没法自动（macOS 安全限制，原因见 [03-撞过的墙.md](03-撞过的墙.md)）。

把访达里高亮的 `feedback-prototype.svg`，**用鼠标拖到 Axure 的画布上松手**。

✅ Axure 画布上出现 3 个反馈页面，文字和形状都可以编辑。

**大功告成！**

---

## 以后每次怎么用

装好后，每次只要 3 行：

```bash
cd ~/Desktop/axure-mac-bridge/svg-generator
node examples/feedback-page.mjs          # 或你自己写的脚本
open -a "$HOME/Library/Application Support/axure-bridge/AxureBridge.app" --args "$HOME/Desktop/原型/feedback-prototype.svg"
```

然后拖一下。权限只在第一次问，后面不再弹。

---

## 排错速查

| 现象 | 原因 | 解决 |
|---|---|---|
| 终端 `command not found: node` | Node 没装/没生效 | 回步骤 0，装完重开终端 |
| `permission denied` | 脚本没执行权限 | `chmod +x INSTALL.sh` |
| AxureBridge 跑完没反应 | 权限对话框被挡 | `⌘+Tab` 找对话框点允许 |
| 点了允许还是不工作 | 权限决定卡住了 | 终端跑 `tccutil reset AppleEvents com.user.axurebridge`，再重试一次重新弹窗 |
| SVG 拖进 Axure 文字歪了 | 你改了脚本但没用 `txt()` | 看 [04-Axure-SVG-怪癖.md](04-Axure-SVG-怪癖.md) |
| Axure 弹"不接受此格式" | 你用了双击打开而不是拖拽 | 必须用鼠标**拖**到画布，不能双击 |
