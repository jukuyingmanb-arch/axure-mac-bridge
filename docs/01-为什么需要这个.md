# 为什么需要 axure-mac-bridge

## 问题陈述

在 macOS 上用 Axure RP 11，**没有任何官方支持的"代码生成原型 → 自动落入 Axure"路径**。

这个矛盾在 Windows 上不存在：
- Windows 版 Axure RP 接受 `Axvg` 格式的 JSON 剪贴板粘贴
- Figma 官方插件 "Axure - Copy Selection for RP" 就是用这个机制
- LLM 可以直接生成 Axvg JSON，调系统剪贴板 + 模拟 Ctrl+V，全自动

但是 Mac 版 Axure RP 用**完全不同的剪贴板架构**：
- 不订阅 `Axvg` UTI
- 复制组件时只在公共 NSPasteboard 放位图 + 一个标记 HTML，真实组件数据在 App 内部 IPC 状态
- 跨 App 粘贴 SVG/JSON/图片都拿不回可编辑的组件，只能得到位图

这意味着在 Mac 上：
- ❌ 没法用 Axvg 剪贴板粘贴
- ❌ 没法用 SVG 剪贴板粘贴（Axure 压扁成位图）
- ❌ 没法用 `open -a "Axure RP 11" file.svg`（Axure 弹"不接受此格式"错误）
- ❌ Axure 菜单里没有"导入 SVG"项（只有"导入 RP 文件"，仅认 .rp 格式）

## 唯一可行的路径：drag-and-drop

Axure for Mac 只接受从 Finder 直接拖文件到画布。SVG 通过 drag-drop 进 Axure 后**可以作为可编辑形状被解析**（如果 SVG 写法符合 Axure 的解析约束，详见 [04-Axure-SVG-怪癖.md](04-Axure-SVG-怪癖.md)）。

## 这个工具链做什么

把"代码生成 SVG → 拖到 Axure"流程里**除最后 1 秒拖动之外的所有步骤都自动化**：

1. **`svg-generator/`**：Node.js 库，提供经过 Axure 解析器适配的 SVG 原语（`txt()` 强制用 `<g transform>` 包裹、手算 text-anchor 偏移等）
2. **`AxureBridge.app`**：macOS 桥接 App
   - 带 `NSAppleEventsUsageDescription` 的 Info.plist，绕过 Claude.app 等没声明此键的父进程被 macOS 静默拒绝 AppleEvents 的问题
   - 自动打开 Finder 到 SVG 所在目录、选中文件
   - 激活 Axure 窗口
   - 让用户**一次拖动即可完成 import**

## 最后那 1 秒为什么绕不开

详见 [03-撞过的墙.md](03-撞过的墙.md)。简短结论：

macOS 安全模型把"用户真实手动拖拽"和"程序合成 mouse 事件"做了底层区分。我们能用 CGEvent 合成完美的 mouseDown→mouseDragged→mouseUp 序列，**但 Finder 不把合成事件认作真 drag**——所以不启动 `NSDraggingSession`、不填充 drag pasteboard，Axure 收不到 `draggingEntered:` 回调。

可能的突破口（本 repo 未实现，欢迎 PR）：
- 自己写 Swift Cocoa app 用 `NSDraggingSession.beginDraggingSession()` 当 drag source
- 用 Hammerspoon 等深层 hook 工具

但 ROI 不值——把"生成 SVG + 找好文件 + 准备 Axure"自动化后，剩下 1 秒手动操作可接受。
