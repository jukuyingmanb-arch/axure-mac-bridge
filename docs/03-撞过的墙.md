# 撞过的墙

为了把"Mac 上 SVG 自动 import 到 Axure"做到 100% 自动，我们尝试了 8 条路径。**全部失败**或**只能做到 99%**。这里是失败档案，给下一位试图突破的人。

## 1. Axvg JSON 剪贴板粘贴

**思路**：Windows 版 Axure 直接支持，Mac 版也应该支持。

**实验**：
- 用 `NSPasteboard.PasteboardType("Axvg")` 写 JSON
- 同时写 `com.adobe.rp.axvg`、各种命名空间变体
- 激活 Axure 发 Cmd+V

**失败原因**：Axure for Mac **完全不订阅** Axvg UTI。复制 Axure 自己的组件时，公共 NSPasteboard 上只有 `public.tiff`（位图）和 `public.html`（含 `(%AXURE-RP-DATA-PRESENTED%)` 字符串的占位 HTML，真实数据在 Axure 内部 IPC 状态）。

**反向采样证据**：在 Axure for Mac 复制一个矩形，dump pasteboard：
```
public.tiff             819 KB（位图）
public.html             152 字符（只是个标记，无真实数据）
NeXT TIFF v4.0 pasteboard type  ...
Apple HTML pasteboard type      ...
```

**结论**：Mac 版 Axure 跨 App 粘贴只能传位图。Axvg JSON 路死。

---

## 2. SVG 文本剪贴板粘贴

**思路**：既然 Axure 支持 SVG 文件 import，可能它也接受 SVG 文本剪贴板。

**实验**：
- 用 5 种 UTI 同时写 SVG 文本：`public.svg-image`、`image/svg+xml`、`public.xml`、`public.utf8-plain-text`、`com.adobe.illustrator.svg`
- 故意不放 PNG/TIFF，强制 Axure 找 SVG
- 用户手动 Cmd+V

**失败原因**：Axure for Mac **不识别 SVG 文本剪贴板**。

**补充**：如果同时写 SVG 和 PNG，Axure 选 PNG（拍扁成位图）。

---

## 3. `open -a "Axure RP 11" file.svg`

**思路**：让 macOS Launch Services 把 SVG 文件交给 Axure。

**失败原因**：Axure 弹错误对话框：
> Valid Axure RP files are: .rp, .rpteam, .rplib, .rpteamlib and .ubx.

Axure 的"打开文件"逻辑只接受 .rp 系列格式。

---

## 4. AppleScript 模拟 File → Import 菜单（**Claude.app 调用**）

**思路**：用 AppleScript UI 脚本点 File → Import → 选 SVG。

**失败原因**：`/Applications/Claude.app/Contents/Info.plist` **缺 `NSAppleEventsUsageDescription` 键**。macOS Mojave (10.14) 起，没声明这个键的 App 想发 AppleEvents 会**静默拒绝**——连权限弹窗都不弹。

报错：`-1743 errAEEventNotPermitted`

`tccutil reset AppleEvents` 都没用，因为问题在 Info.plist 不在权限决定。

**绕过办法**：自建一个声明了这个键的中介 .app（就是这个 repo 的 `AxureBridge.app`）。

---

## 5. AppleScript 模拟 File → Import 菜单（**AxureBridge.app 调用**）

**思路**：权限通了（#4 的绕过），应该能成功。

**实验**：成功导航到 Axure File 菜单，发现唯一含"导入"的菜单项是 **"导入RP文件(I)..."**——只接受 .rp 格式。继续 dump 全部菜单（Apple、Axure RP 11、文件、编辑、视图、项目、布局、发布、团队、账号、窗口、帮助）。

**失败原因**：**Axure for Mac 整个菜单系统没有"导入 SVG"或"导入图像"选项**。它做了 export 但没做 import。

这是 Axure for Mac 的产品决策：Mac 版只支持 drag-drop import。

---

## 6. 文件 URL 剪贴板 + CGEvent Cmd+V

**思路**：把文件作为 file URL 放剪贴板（像 Finder 复制文件那样），然后让 Axure paste。

**实验**：
- Swift 用 `NSPasteboard.writeObjects([NSURL])` 写文件 URL
- 验证 `public.file-url` 和 `NSFilenamesPboardType` 都在剪贴板上
- 用 CGEvent（绕过 AppleScript 权限）合成 Cmd+V

**失败原因**：Axure for Mac 的 paste 处理器**不响应 file URL paste**。`pasteboard.types` 里有 file-url 但 Axure 不读。

---

## 7. CGEvent 鼠标拖拽合成

**思路**：用 CGEvent 合成从 Finder 文件 icon 到 Axure 画布的完整 drag 鼠标序列。理论上 Finder 会把 mouseDown + mouseDragged × N + mouseUp 序列识别为真实拖拽，启动 `NSDraggingSession` 把 file URL 放到 drag pasteboard，Axure 收到 `draggingEntered:` → `performDragOperation:`。

**实验**：
- Finder 用 AppleScript 定位到文件 icon 屏幕坐标 (530, 383)
- Axure 用 AppleScript 拿画布中心 (2472, 510)
- Swift 用 `CGEvent(mouseEventSource:...)` 合成：
  - `mouseMoved` 到源点
  - `leftMouseDown`
  - 5 次 `leftMouseDragged` 在源点附近抖动（超过 macOS drag 阈值）
  - 20 次 `leftMouseDragged` 缓慢移动到目标
  - `leftMouseUp` 在目标
- 全部 `.cghidEventTap` post

**失败原因**：**macOS 把合成 CGEvent 和真实硬件输入做了底层区分**。Finder 收到合成事件，但**不**启动 `NSDraggingSession`，drag pasteboard 保持空状态。

合成的 mouseUp 落在 Axure 上看起来像 click 不像 drop，所以 Axure 啥也不做。

参考：CGEvent 的 `kCGEventSourceStateID` 不同（`.hidSystemState` vs `.combinedSessionState`）有微妙差异，但都不能伪装成真实输入触发 drag session。

---

## 8. 完全自动化的最后突破口（未实施）

剩下两条**理论上可行但未尝试**的路：

### A. 自写 Swift drag source App

```swift
// 在 Cocoa app 里：
let view: NSView = ...                    // 任意 view
let item = NSDraggingItem(pasteboardWriter: fileURL as NSURL)
let event = NSEvent(mouseEvent: ...)      // 合成 mouseDown
view.beginDraggingSession(with: [item], event: event, source: ...)
```

如果系统接受合成的 NSEvent 启动 drag session，drag pasteboard 就被正确填充。然后用 CGEvent 把光标移到 Axure 画布发 mouseUp，drop 发生。

风险：`NSEvent(mouseEvent:...)` 构造的合成事件可能被 NSDraggingSession 内部检测并拒绝。

### B. Hammerspoon 或类似工具

[Hammerspoon](https://www.hammerspoon.org/) 用 Lua 提供系统级 hook，`hs.eventtap.event` 能发更底层事件。社区里有人用它做 drag-drop 自动化。

代价：用户需要装第三方工具。

---

## 经验总结

1. **macOS 安全模型把跨 App 自动化分了 N 层权限**。每层都可能挡你：
   - Accessibility（合成鼠标键盘）
   - Automation（AppleEvents 到特定 App）
   - Info.plist usage descriptions
   - sandbox file access
   - 合成 vs 真实输入区分
2. **`tccutil reset AppleEvents` 不是万能**。如果 App 的 Info.plist 缺 usage description，重置没用——根因不在权限决定。
3. **`set frontmost of process` AppleScript 不够强**，常常不能让 dock 里的窗口冒出来。要用 `tell application X to activate`。
4. **macOS Bash 3.2 的 `$()` 命令替换里 heredoc 处理有 quote 怪癖**，单引号撇号会让 parser 错乱（"isn't" 类型的注释会炸）。
5. **沙箱化的 Claude.app 等 IDE/Chat 应用可以写 `~/Desktop/` 但 `ls ~/Desktop/` 被拒**——只写不读。
