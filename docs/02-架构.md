# 架构

## 数据流

```
你的描述/需求
       ↓
   [人脑或 LLM]
       ↓
  Node.js 生成脚本（用 svg-generator/lib 提供的原语）
       ↓
   SVG 文件（~/Desktop/原型/<name>.svg）
       ↓
   ┌─────────────────────────────────┐
   │  open -a Safari ...svg          │ ← 用户预览
   │  open -a AxureBridge.app ...    │ ← 准备 Finder + Axure
   └─────────────────────────────────┘
       ↓
   Finder 选中文件 + Axure 激活
       ↓
   [用户 1 秒拖动] ← 唯一手动步骤
       ↓
   Axure 画布上出现可编辑形状
```

## 模块职责

### svg-generator/

**核心约束**：所有产物必须能被 Axure for Mac 的 SVG 解析器正确识别。

**`lib/primitives.mjs`** — SVG 基础原语：
- `txt(x, top, str, opts)`：文字。**必须**用此函数，不要直接写 `<text>`
  - 内部自动包 `<g transform="translate(...)">`（绕过 Axure 吃 y 坐标的 bug）
  - `text-anchor` 被 Axure 忽略，所以 `anchor: 'middle' | 'end'` 通过预先用 `measureText()` 算偏移实现
  - y 是"视觉顶边"，内部加 `size × 0.8` 转成基线 y
- `measureText(str, size)`：字符宽度估算（CJK ≈ 1.0×size，ASCII ≈ 0.55×size）
- `rect(x, y, w, h, fill, opts)`：矩形
- `txtBlock(...)`：多行文字（按字符宽度自动断行）
- `colors`：标准配色

**`lib/phone.mjs`** — 移动端 mockup 元件：
- `phoneFrame(ox, oy, label)`：手机外框 + 阴影 + 顶部标签
- `statusBar()`：iPhone 状态栏
- `navBar(title, rightLabel?)`：导航栏
- `homeIndicator()`：底部小条
- `pill(...)`：圆角胶囊（chip / tag）

**`lib/spec-panel.mjs`** — 右侧需求说明面板：
- `specPanel(ox, oy, pageNum, title, sections)`：渲染一个完整的 spec panel
  - 支持 `•` 一级、`-` 二级缩进的层级列表
  - 自动换行

### AxureBridge.app/

**核心目的**：提供一个**有自己 bundle ID + NSAppleEventsUsageDescription 声明**的 macOS App，让 AppleScript 自动化能从一个已授权的源运行。

为什么需要：
- Claude.app 等 IDE/Chat 应用通常不声明 NSAppleEventsUsageDescription
- 缺这个键时 macOS 会**静默拒绝**所有 AppleEvents（连权限请求弹窗都不弹）
- 自建一个声明了这个键的 .app 绕过这道墙

**Info.plist** 关键字段：
- `CFBundleIdentifier = com.user.axurebridge`
- `NSAppleEventsUsageDescription = ...`（必须，否则 macOS 默拒）
- `LSUIElement = true`（无 Dock 图标，是 background helper）

**MacOS/AxureBridge**（主可执行 bash 脚本）：
- 接收 SVG 路径（命令行参数 `--args` 或 `/tmp/axure-bridge-input` 文件）
- 根据 `/tmp/axure-bridge-mode` 选模式（默认 `prepare`）
- 调对应的 AppleScript

**Resources/** 里的 4 个脚本：
| 文件 | 模式 | 用途 |
|---|---|---|
| `prepare-drag.applescript` | `prepare`（默认）✅ | 打开 Finder + 选中 SVG + 激活 Axure |
| `dump-all-menus.applescript` | `dump` 🔧 | 诊断：枚举 Axure 全部菜单 |
| `import.applescript` | `menu-import` ⚠️ | 试 File → Import（只对 .rp 有效） |
| `drag-import.applescript` + `drag-cgevent.swift` | `drag` ⚠️ | CGEvent 拖拽合成（不工作，保留参考） |

## 调用入口

最简单的全流程：

```bash
node svg-generator/examples/feedback-page.mjs

open -a Safari ~/Desktop/原型/feedback-prototype.svg

open -a "$HOME/Library/Application Support/axure-bridge/AxureBridge.app" \
     --args "$HOME/Desktop/原型/feedback-prototype.svg"
```

可以包成一个 shell 脚本或在生成脚本末尾用 `child_process.spawn` 串起来。

## 权限模型

3 道独立的 macOS 权限：

1. **Accessibility**（系统设置 → 隐私 → 辅助功能）
   - 给 `AxureBridge.app`
   - 用于 osascript 读窗口位置、合成键盘事件
2. **Automation: System Events**（系统设置 → 隐私 → 自动化）
   - 给 `AxureBridge.app` 控制 `System Events`
   - 用于 UI Scripting 菜单导航
3. **Automation: Axure RP 11 / Finder**
   - 给 `AxureBridge.app` 控制这些 App
   - 用于激活窗口、操作 Finder

首次运行 AxureBridge 会逐个弹窗请求，**点允许**即可。
后续运行不再弹。

如果某个权限被误拒，用 `tccutil reset AppleEvents com.user.axurebridge` 重置。
